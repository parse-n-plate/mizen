Parse-n-Plate: Technical Summary for CTO
========================================

Project Overview
----------------
Parse-n-Plate is a web application that extracts and displays clean, ad-free recipes from any cooking website. Users input a recipe URL, and the system parses the recipe content, removing ads and unnecessary elements to provide a clean recipe view.

Architecture Overview
--------------------
High-Level Architecture:
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   Frontend      │    │   Backend APIs   │    │   External      │
│   (Next.js)     │◄──►│   (Next.js API)  │◄──►│   Services      │
│                 │    │                  │    │                 │

Tech Stack
----------

Frontend:
- Framework: Next.js 15.3.4 (React 19)
- Language: TypeScript
- Styling: Tailwind CSS v4
- UI Components: 
  - Radix UI (accessible components)
  - Shadcn/ui component library
  - Lucide React (icons)
- State Management: React Context API
- Build Tool: Next.js built-in bundler

Backend:
- Runtime: Node.js 24.1
- Framework: Next.js API Routes
- Language: TypeScript
- HTTP Client: Axios
- HTML Parsing: Cheerio

Python Integration:
- Python Version: 3.9+
- Key Libraries:
  - recipe-scrapers: Specialized recipe extraction
  - beautifulsoup4: HTML parsing
  - requests: HTTP requests
- Integration: Child process spawning from Node.js

AI/ML:
- Provider: Groq AI
- Model: llama3-8b-8192 (default)
- Fallback Model: mistral-saba-24b (requires terms acceptance)
- Use Case: Recipe parsing when Python scraper fails

Infrastructure & DevOps:
- Containerization: Docker
- Cloud Platform: AWS
- Container Registry: Amazon ECR
- Orchestration: Amazon ECS (EC2 launch type)
- CI/CD: GitHub Actions
- Reverse Proxy: Nginx (on EC2)
- Region: us-west-1

Data Flow
---------
Recipe Parsing Pipeline:
1. URL Validation: Checks if URL contains recipe content
2. Primary Parsing: Python script with recipe-scrapers library
3. Fallback Parsing: AI-powered parsing using Groq API
4. Data Processing: Format standardization and cleaning
5. Storage: React Context + localStorage
6. Display: Clean, formatted recipe view

API Endpoints:
- POST /api/urlValidator - Validates recipe URLs
- GET /api/fetchHtml - Fetches and cleans HTML content
- POST /api/recipeScraperPython - Python-based recipe scraping
- POST /api/parseIngredients - AI-powered ingredient extraction
- POST /api/parseInstructions - AI-powered instruction extraction
